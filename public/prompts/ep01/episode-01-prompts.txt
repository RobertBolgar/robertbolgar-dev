EPISODE 1
BUILDING ROBERTBOLGAR.DEV
==================================================

PROJECT OVERVIEW

Episode 1 documents the initial creation of robertbolgar.dev as a single-page React + Vite website. The site serves as the public home base for Robert Bolgar's product-building journey, featuring sections for videos, projects, devlog, and about information. The design follows a premium public-building media brand aesthetic inspired by Vercel, Linear, and Stripe.

Tech Stack:
- React with functional components
- Vite as build tool
- Plain CSS with CSS variables
- Google Fonts: Bebas Neue, DM Sans, DM Mono
- No UI frameworks, CMS, or extra pages

==================================================
LESSONS LEARNED
==================================================

1. Node.js Compatibility
- Initial Vite version was incompatible with Node.js 22.11.0
- Solution: Downgraded Vite to 5.4.11 and @vitejs/plugin-react to 4.3.4
- Always check Node.js version compatibility before installing dependencies

2. Design Hierarchy
- Typography hierarchy is critical for premium feel
- Hero should be largest, section headings second, card titles third
- Consistent spacing between sections creates breathing room
- Subtle background effects add depth without visual clutter

3. Alignment and Centering
- Centering section headings and intro text creates balance
- Card grids need max-width containers to prevent awkward layouts
- Mobile responsiveness requires explicit media query adjustments
- Using margin: 0 auto is cleaner than hard-coded margins

4. AI Search Optimization
- JSON-LD schema helps search engines understand content structure
- Canonical URLs prevent duplicate content issues
- Third-person language ("Robert Bolgar" vs "I") improves clarity for AI systems
- Structured llms.txt with clear sections aids LLM understanding

5. Version Control
- .gitignore should always include node_modules, dist, and .env files
- Public repositories need careful attention to sensitive data
- Commit messages should be descriptive but concise
- GitHub integration enables automatic deployment workflows
